# 数据真实性证明指南

## 📋 概述

本文档说明如何向老师证明项目数据的真实性和完整性。

---

## ✅ 已实施的数据验证机制

### 1. **实时数据签名系统**
- **算法**: FNV-1a 哈希算法
- **包含内容**: 
  - 所有数据点的完整信息（频率、幅值、相位）
  - 会话 ID（每次启动唯一）
  - 实时时间戳
  - 数据点索引和计数
- **签名示例**: `FNV1a-A3F2B8C1`
- **作用**: 任何数据被篡改都会导致签名不匹配

### 2. **唯一会话标识**
- **格式**: `时间戳36进制-随机数`
- **示例**: `LQZX8M9-7A3F2BC`
- **作用**: 每次运行程序都有唯一标识，防止数据复制

### 3. **系统环境指纹**
- 浏览器信息
- 操作系统平台
- Web Serial API 特征
- 实时时间戳

### 4. **增强型 CSV 导出**
导出的 CSV 文件包含：
```
# 频率响应分析仪 - 数据导出文件
# ============================================
# 导出时间：2025/11/27 19:15:23
# ISO时间戳：2025-11-27T11:15:23.456Z
# 设备型号：GD32F103CBT6
# 固件版本：v2.0.1
# 通信协议：串口 115200bps
# 浏览器：Chrome/119.0.0.0
# 操作系统：MacIntel
# 数据点数：100
# 数据签名：FNV1a-A3F2B8C1
```

## 🎥 建议的演示流程

### 方案一：完整录屏演示（强烈推荐）
1. **开始录屏**（使用 QuickTime/OBS 等工具）
2. **打开网页** - 显示初始空白状态
3. **连接硬件** - 展示串口选择对话框
4. **实时采集** - 让老师看到：
   - 数据一条条实时出现
   - 图表动态更新
   - 日志窗口实时刷新
   - 校验码动态变化
5. **展示验证区域** - 特写"数据验证与溯源"部分
6. **导出数据** - 演示 CSV 导出并打开文件
7. **结束录屏**

**优势**: 无法造假，时间戳连续，数据实时生成

### 方案二：现场演示
1. 带着硬件设备和电脑到老师办公室
2. 当场连接、采集、展示
3. 让老师亲眼看到整个过程

**优势**: 最具说服力

### 方案三：分段证明
1. **硬件照片** - 拍摄 GD32 开发板实物
2. **串口截图** - 显示设备管理器中的 USB 串口
3. **过程录像** - 从连接到数据采集的完整视频
4. **数据文件** - 提交带完整签名的 CSV 文件

## 📱 额外验证手段

### 1. 使用手机录制屏幕
- 用手机拍摄电脑屏幕
- 可以看到鼠标操作、实时数据变化
- 无法事后编辑

### 2. 直播串流
- 使用腾讯会议/Zoom 分享屏幕并录制
- 云端保存，时间戳可验证

### 3. Git 提交记录
- 提交代码到 GitHub
- 提交历史显示开发过程
- README 中包含演示视频链接

### 4. 数据对比验证
- 多次运行，每次生成不同的：
  - 会话 ID
  - 数据签名
  - 时间戳
- 证明数据是实时生成的

## 📊 数据文件命名规范

导出的文件名格式：
```
frequency_response_2025-11-27T11-15-23_A3F2B8C1.csv
                    ↑                   ↑
                  时间戳              签名前8位
```

这样老师可以：
- 从文件名看到创建时间
- 验证签名一致性
- 区分不同批次的数据
